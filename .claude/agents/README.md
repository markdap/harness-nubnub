# 서브에이전트 (Sub-Agents)

서브에이전트는 특정 반복 작업이나 전문 역할을 위임할 때 만든다.
프로젝트마다 필요할 수도, 불필요할 수도 있다. 필요할 때 만들면 된다.

## 언제 만드는가

- 특정 역할을 반복적으로 수행해야 할 때 (예: 코드 리뷰어, 데이터 검증자)
- 메인 에이전트가 할 일이 너무 많아 역할을 분리해야 할 때
- 특수한 도메인 지식이 필요한 작업 (예: 보안 검토, 성능 분석)

## 만드는 방법

1. `_template.md`를 복사한다
2. 파일명을 역할에 맞게 바꾼다 (예: `code-reviewer.md`, `data-validator.md`)
3. 템플릿의 플레이스홀더를 채운다

## 운영 방식

- 서브에이전트는 메인 에이전트의 지시에 따라 동작한다
- 서브에이전트는 자신의 역할 범위 밖의 작업은 하지 않는다
- 메인 에이전트가 서브에이전트에게 작업을 위임하고 결과를 받는다
