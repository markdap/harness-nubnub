감독님이 `/handoff`를 요청하면 아래 두 단계를 순서대로 실행하라.

---

## 1단계: 문서 최신화 검증

`docs/` 하위 파일들을 읽고 현재 구현 상태와 비교하라.

체크 항목:
- **PRD.md**: 핵심 기능 목록이 실제 구현된 기능과 일치하는가? 제외 사항이 여전히 유효한가?
- **ARCHITECTURE.md**: 디렉토리 구조·패턴이 실제 코드베이스를 반영하고 있는가?
- **ADR.md**: 이번 세션에서 내린 결정 중 기록되지 않은 것이 있는가?

오래된 내용이 있으면 해당 파일을 수정하고 커밋한다:
```
docs: update docs for session handoff
```

변경이 없으면 "문서 최신 상태 확인됨"을 보고하고 다음 단계로 넘어간다.

---

## 2단계: HANDOFF.md 작성

`docs/HANDOFF.md`를 작성하라. **다음 세션의 AI가 읽고 30초 안에 현재 상태를 파악할 수 있어야 한다.**

필수 포함 항목:
1. **현재 Phase/Step** — `phases/index.json`을 읽어 정확한 phase명과 step 번호를 기록
2. **완료된 것** — 이번 세션에서 완료한 작업을 구체적으로 열거
3. **다음 할 일** — 새 세션에서 바로 이어서 할 첫 번째 액션 (모호하지 않게)
4. **결정 사항** — 이번 세션의 기술/기획 결정과 그 이유
5. **주의사항** — 막혔던 지점, 피해야 할 것, 알아두어야 할 특이사항

작성 후 커밋:
```
docs: session handoff [날짜]
```
